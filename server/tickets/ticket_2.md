**Relationships (summary)**

* **One category → many products:** `products.category_id` → `categories.id` (`ON DELETE RESTRICT`).
* **One product → many variants:** `product_variants.product_id` → `products.id` (`ON DELETE CASCADE`).
* **One product → many reviews:** `reviews.product_id` → `products.id` (`ON DELETE CASCADE`). `reviews.user_id` has **no foreign key** (no `users` table in this ticket); it stores the logical user id for when you add users later.

**Product cards** come from joining `products` + `categories` (and optionally `product_variants` / `reviews` in queries), not a separate table.
